## Gatsby parses queries in React components

- Use (local) schema to query against
- Parse queries, and inject components with data
- No run-time cost _at all_ for this
- e.g. you don't hit the CMS client-side

---

## Let's take a look

---

export default FullScreenCode;

```jsx
import React from 'react';
import { graphql } from 'gatsby';

export default function IndexPage({ data }) {
  return <h1>{data.site.siteMetadata.title}</h1>;
}

export const indexQuery = graphql`
  {
    site {
      siteMetadata {
        title
      }
    }
  }
`;
```

---

- The shape _matches_ the data
- The query is compiled away (no run time cost!)
- Static HTML is produced (ðŸŽ‰)
